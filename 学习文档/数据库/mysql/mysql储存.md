## mysql的储存原型
```
1. 数据库
	 就是在mysql中 data文件夹下的 文件夹
2. 表
	以 frm 结尾
		表结构
	以 MYD 
		表数据
	以 MYI
		表索引
3. 储存类型比较
	MYISAM 
		是以指针储存
	InnoDB
		是以行数据储存
4. 聚集索引和非聚集索引
	1. 聚集索引就是包含了完整的数据记录，所以 InnoDB就是一个 聚集索引，而MYISAM是一个 非聚集索引（还要根据指针加载数据文件）

5. 为什么InnDB表必须有主键，并且推荐使用整形的自增主键？
	因位B+树，是从左到右逐渐变大，如果主键不自增，就会出现分裂，出现重组，耗费性能

6.B+ 树与hash 的比较
	1. hash 对于单个查询 （=） 查询一次就可以查询到对应数据的指针，缺点是： 范围查询不支持
	2. B+树 因为是从左到右 递增的，所以支持范围查询
```
